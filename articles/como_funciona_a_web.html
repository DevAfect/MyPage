Sobre a web e a estrutura dos sites

Como funciona a Web ?

Bom, para entender como funciona WEB precisamos pensar na sua estrutura, na internet em si, que na verdade nada mais é que um empilhado de computadores e cabos que conectam todo o mundo, seguindo uma teoria bem legal de redes que afirma que todos estão a seis graus de separação (as vezes um pouco mais que isso).

Podemos olhar tudo de uma maneira bem simples e pensar que o computador (cliente) pede para um determinado um determinado host (servidor) dados. Existem diversas etapas para que essa comunicação seja feita e todo um método especifico do como se comunicação. Mas falando grosseiramente é como se o navegador do computador "fala-se através da internet" para um site. Isso pode ser entendido como se ele estivesse solicitando os dados contidos naquele endereço, vamos chamar de endereço eletrônico (URL), em seguida o servidor, que contém os dados pedidos, os envia para o navegador e lá são exibidos!

Mas como um computador pede e recebe uma página?

Para pedir uma página o cliente do computador segue o Protocolo de Transferência de Hipertexto (HTTP) e isso serve como base para toda comunicação, os métodos de solicitação são:

  • REQUEST
    • GET
    • HEAD
    • PUT
    • POST
    • PATCH
    • TRACE
    • options
    • delete
  • RESPONSE

Com esses métodos é possivel realizar toda comunicação que vemos nos sites! tudo isso tem por base a ideia que o cliente envia uma mensagem de requisição e o servidor envia uma mensagem de resposta ao cliente com aquilo que foi pedido.

Um navegador pedindo dados de um servidor local, seria algo como:

GET / HTTP/1.1
Host: localhost:8080
Connection: keep-alive
sec-ch-ua: " Not A;Brand";v="99", "Chromium";v="101"
sec-ch-ua-mobile: ?0
sec-ch-ua-platform: "Linux"
DNT: 1
Upgrade-Insecure-Requests: 1
User-Agent: Mozilla/5.0 (X11; Linux x86_64) AppleWebKit/537.36 (KHTML, like Gecko) Chrome/101.0.4951.64 Safari/537.36
Accept: text/html,application/xhtml+xml,application/xml;q=0.9,image/avif,image/webp,image/apng,*/*;q=0.8,application/signed-exchange;v=b3;q=0.9
Sec-Fetch-Site: none
Sec-Fetch-Mode: navigate
Sec-Fetch-User: ?1
Sec-Fetch-Dest: document
Accept-Encoding: gzip, deflate, br
Accept-Language: pt-BR,pt;q=0.9,en;q=0.8,en-US;q=0.7

Os navegadores usuais (chrome, firefox, edge) são clientes muito complexos e carregam em si uma arquitetura gigantesca, analisando o pedido de um cliente menos complexo, o curl, podemos ver a seguinte resposta: 

GET / HTTP/1.1
Host: localhost:8080
User-Agent: curl/7.83.1
Accept: */*

Essa é uma solicitação típica, a primeira linha tem o GET que é o método solicitado, a versão http e cada linha depois disso é um valor chave de cabeçalho.


Host : O servidor a qual está sendo feito o pedido.
User-Agent: a identificação do cliente que está pedindo.
Accept: o tipo de arquivo que é esperado de volta.
E dois espaços em branco.

Fazendo um pedido a URL, utilizando o curl https://www.infnet.edu.br/infnet/home/ temos o seguinte como resposta 

HTTP/1.1 301 Moved Permanently
Server: nginx
Date: Sun, 30 Oct 2022 01:25:29 GMT
Content-Type: text/html
Content-Length: 162
Connection: keep-alive
Location: https://www.infnet.edu.br/infnet/home/

Podemos ver que na resposta o servidor envia muitas coisas, entre elas podemos notar o "Content-Type" e ali temos o conteúdo da página que é renderizada no navegador!

Mas como o computador fala com a internet ?

Para isso precisamos entender como funciona o "Domain Name System" (DNS), que funciona como a lista telefônica da internet, é com isso que ela sabe como procurar o endereço de um site, ela traduz o "www.xablau.com" em uma sequência de numeros chamada endereço de IP e com base no nome que foi pedido que ela faz todo o caminho.

Para fazer o caminho existe toda uma estrutura hierarquica e distribuída com raizes que tem suas proprias zonas que fazem tudo isso funcionar, mas eu escrevo sobre isso quanto tiver mais tempo.

O que é uma Página WEB?

Uma página web é uma coisa linda, é uma estrutura criada com base em uma Linguagem de Marcação de Hipertexto no caso o HTML, que possibilita aos navegadores lerem estas marcações e traduzirem o que foi indicado, manipulando texto, imagem, links e varios outros conteúdos que são exibidos na página.

Qual a função da linguagem HTML e CSS na página WEB ?

O HTML é a linaguem de marcação com base em tags que faz as indicações do que deve ser feito em cada parte da página, enquanto o CSS (Cascading Style Sheets ou Folhas de Estilo em Cascata) é o arquivo de estilização das tags da página, nele é descrito como vai ser apresentado de cada o conteúdo das tags

O que é o ambiente de programação para desenvolver HTML/CSS ?

Em geral são softwares ou serviços web que permitem a integração de diversas ferramentas para programar um site! São facilitadores que economizam o tempo dos programadores, fornecendo diversas ferramentas que agilizam o trabalho
